## Cómo Invocar

```
/frontend-design [tarea]
/backend-design [tarea]
/ui-ux-pro-max [tarea]
```

O simplemente menciona el skill en tu pregunta.

---

## Flujo Típico

1. **Generar Design System**: `/ui-ux-pro-max Generar sistema para RK Solutions`
2. **Crear Componente Frontend**: `/frontend-design Crear hero section`
3. **Implementar Backend**: `/backend-design Crear handler de demo requests`
4. **Integrar**: Conectar frontend con backend
5. **Validar**: Probar en múltiples dispositivos

## Checklists Clave

### Antes de Completar Cada Componente

✅ **Frontend**
- [ ] HTML semántico
- [ ] CSS con design system
- [ ] Responsive (375px, 768px, 1024px)
- [ ] Accesibilidad WCAG AA
- [ ] Hover states definidos
- [ ] Focus states visibles

✅ **Backend**
- [ ] Input validado
- [ ] Datos sanitizados
- [ ] Prepared statements
- [ ] Error handling
- [ ] Logging seguro
- [ ] API responses JSON

✅ **Design System**
- [ ] Colores consistentes
- [ ] Tipografía uniforme
- [ ] Spacing 8px grid
- [ ] Sombras consistentes
- [ ] Transiciones suaves
- [ ] Anti-patrones evitados

## Tips de Uso Efectivo

1. **Lee el brief primero** - Contextualiza con RK_WEB_BRIEF.md
2. **Sé específico** - Define exactamente qué necesitas
3. **Usa checklists** - Verifica calidad antes de completar
4. **Documenta** - Mantén código comentado en backend
5. **Prueba múltiples dispositivos** - No solo desktop
6. **Valida accesibilidad** - WCAG 2.1 AA es obligatorio
